# AGENTS.md

本文件包含了专为 AI Agent (Gemini CLI) 设计的特定工作流和开发准则。

> [!IMPORTANT]
> **在进行深度开发或修改核心逻辑前，请先查阅 [ARCHITECT.md](./ARCHITECT.md) 以了解系统整体架构、模块职责及关键工作流。**

## 核心开发准则

1.  **语言偏好**: 所有思考过程、计划过程、以及与用户的对话均使用**简体中文**。
2.  **环境隔离**: Python 开发**必须**在 `.venv` 虚拟环境中进行。在执行任何脚本、安装依赖或运行测试前，必须确保虚拟环境已激活。
3.  **代码风格与质量**: 
    - 保持代码简洁、高效且符合行业标准。
    - 所有核心逻辑、公共方法和重要变更必须附带恰当的中文注释。
    - **强制要求**: 每次执行单元测试前，**必须**同步执行代码规范检查 (`ruff check`) 与格式化检查 (`ruff format`)。
4.  **安全性与完整性**:
    - 严禁泄露、打印或提交任何敏感信息（如 API 密钥、私钥、凭据等）。
    - 始终遵循安全编码最佳实践。
    - 遵循 RESTful API 设计规范。

## 开发工作流

在进行任何代码修改前，请务必执行以下步骤：

1.  **阅读上下文**: 完整阅读本文件 (`AGENTS.md`)。
2.  **更新文档**: 
    - 每次完成重大功能开发或里程碑后，必须同步更新 `AGENTS.md`。
    - 每次修改项目技术栈时，必须更新 `AGENTS.md`。
3.  **Git 操作**:
    - 除非用户明确要求，禁止执行任何 Git 提交 (`git commit`) 或推送 (`git push`) 操作。
    - 在执行 Git 操作前，若 `AGENTS.md` 已有更新，必须确保该文件已被包含在提交中。
